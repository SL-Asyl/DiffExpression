## Автоматизированный пакетный анализ обогащения генных наборов (ORA: KEGG, GO, MP) на основе данных дифференциальной экспрессии

### 1. Программное окружение и автоматизация зависимостей

Анализ реализован на языке программирования R. Скрипт снабжен блоком динамического контроля зависимостей, который при каждом запуске проверяет наличие необходимых библиотек в локальной библиотеке R и автоматически производит их установку в случае отсутствия.

-   **Репозиторий CRAN**: Проверяется и устанавливается базовое вычислительное и инфраструктурное окружение: openxlsx (взаимодействие с форматом Excel), enrichR (интерфейс к базам данных фенотипов), dplyr и tidyr (манипуляции с матрицами данных высокого порядка).

-   **Репозиторий Bioconductor**: С помощью менеджера BiocManager разворачиваются специализированные биоинформатические пакеты: clusterProfiler (ядро статистического ORA-тестирования), AnnotationDbi и org.Mm.eg.db (база данных маппинга и аннотации генов Mus musculus), а также транскриптомные сигнатурные пакеты ReactomePA и rWikiPathways.

### 2. Подготовка данных и фильтрация фонового генного списка (Background Universe)

Пакетная обработка запускается в циклическом режиме для всех файлов дифференциальной экспрессии (.xlsx), находящихся в рабочей директории raw_data. Для каждого файла (соответствующего отдельному клеточному кластеру или типу клеток) последовательно выполняются следующие этапы:

-   **Очистка матриц**: Из исходного датасета извлекаются три ключевые колонки: официальные символы генов (names), уровень изменения экспрессии (logfoldchanges) и скорректированный уровень статистической значимости (pvals_adj). Строки, содержащие неопределенные значения (NA) в любой из этих колонок, необратимо удаляются.

-   **Конвертация идентификаторов (ID Mapping)**: Поскольку базы данных путей оперируют числовыми кодами NCBI, выполняется маппинг официальных символов генов (Gene Symbols) в идентификаторы Entrez ID с помощью пакета org.Mm.eg.db. Параметр multiVals = "first" гарантирует строгое однозначное соответствие: в случае неоднозначного маппинга берется первый верифицированный Entrez ID.

-   **Формирование «Вселенной генов»**: Гены, не получившие валидного Entrez ID, исключаются. Оставшийся пул уникальных идентификаторов фиксируется как индивидуальный фоновый список (universe_genes) для данного типа клеток. Это критически важно для корректного расчета статистики гипергеометрического распределения.

### 3. Отбор дифференциально экспрессирующихся генов (DEGs)

Для изоляции таргетных биологических сигналов из очищенного фонового списка формируются четыре независимых вектора генов на основе порогов статистической значимости ($FDR < 0.05$) и биологического эффекта ($|log_2FC| \ge 1.0$):

-   **Up-regulated (Повышенная экспрессия)**: \* Entrez ID вектор (up_genes): $pvals\_adj < 0.05$ и $logfoldchanges \ge 1.0$.
    -   Gene Symbol вектор (up_symbols): аналогичные пороги для интерфейса Enrichr.
-   **Down-regulated (Подавленная экспрессия)**:
    -   Entrez ID вектор (down_genes): $pvals\_adj < 0.05$ и $logfoldchanges \le -1.0$.
    -   Gene Symbol вектор (down_symbols): аналогичные пороги для интерфейса Enrichr.

**Критерий контроля качества (QC Fail-Safe)**: Перед запуском математического ядра ORA скрипт проверяет длину каждого вектора. Если количество значимых генов в пуле составляет меньше 5, расчет для данного направления экспрессии блокируется с выводом предупреждения в системную консоль. Это предотвращает генерацию ложноположительных артефактов на малых выборках.

### 4. Статистическое тестирование избыточного представительства (ORA)

Запуск ORA выполняется параллельно по трем независимым онтологическим базам данных. В качестве статистического метода используется односторонний гипергеометрический тест (фишеровский точный тест), оценивающий, является ли присутствие генов из списка DEGs в конкретном биологическом пути более частым, чем ожидалось бы при случайном выборе из фоновой «вселенной».

-   **KEGG Pathways**: Анализ проводится через функцию enrichKEGG для организма Mus musculus (organism = "mmu"). Порог $p$-value установлен на уровне $< 0.05$, порог $q$-value (мера ложного обнаружения с учетом вложенности путей) ограничен как $< 0.20$.

-   **Gene Ontology (GO)**: Анализ охватывает одновременно три домена онтологии: биологические процессы (BP), клеточные компоненты (CC) и молекулярные функции (MF) при помощи enrichGO (ont = "ALL"). Параметр readable = TRUE активирует внутренний обратный маппинг, переводящий числовые Entrez ID в финальной таблице в читаемые Gene Symbols.

-   **Mammalian Phenotype (MP)**: Анализ ассоциации генов с фенотипическими отклонениями млекопитающих выполняется через облачный API enrichR по двум репозиториям консорциума MGI: MGI_Mammalian_Phenotype_Level_4_2021 и Level_3_2021.

Во всех тестах контроль множественного тестирования и расчет $p.adjust$ осуществляются по методу Бенджамини-Хохберга (BH / FDR).

### 5. Алгоритмическая реструктуризация и денормализация данных

Стандартный вывод ORA группирует результаты по ID биологического пути, записывая все гены, попавшие в этот путь, в одну ячейку текстовой строкой через разделитель (/ или ;). Для превращения этих сырых матриц в реляционную структуру, пригодную для создания сводных Мастер-таблиц, применяется кастомная функция трансформации:

-   **Парсинг и денормализация строк**: Строка генов расщепляется по регулярному выражению разделителя. С помощью операции separate_rows таблица подвергается структурному «взрыву» — строка пути дублируется для каждого гена отдельно, формируя длинный (long-format) формат матрицы.

-   **Пересчет метрик**: Значения числовых соотношений (GeneRatio и BgRatio) переводятся из дробных строк в проценты с округлением до одного знака после запятой (например, 14.5%). Значения статистической значимости (pvalue, p.adjust, qvalue) приводятся к единому научному формату с фиксацией 3 значащих цифр (signif(x, 3)).

-   **Группировка и сортировка**: Колонка индивидуальных символов генов (Genes) принудительно переносится на первую позицию (первый столбец датафрейма). Итоговая таблица сортируется по алфавиту по имени гена, а внутри одного гена — по уровню значимости биологического пути ($p$-value от самых репрезентативных к менее значимым).

### 6. Многопоточный экспорт результатов

Для каждого обработанного типа клеток автоматически формируется индивидуальный файл Excel, защищенный от перезаписи результатов других кластеров с помощью уникального префикса: ORA_results/ORA_KEGG_GO\_[Cell_Type].xlsx.

Внутри файла структура организуется по вкладкам:

-   KEGG_UP / KEGG_DOWN: Вывод денормализованных таблиц путей KEGG.

-   GO_MP_UP / GO_MP_DOWN: Комплексные листы, где результаты GO и фенотипов MP записываются строго друг под другом вертикальными блоками. Каждый блок предваряется уникальным текстовым заголовком (например, "GO enrichment (UP)" или "Mammalian Phenotype enrichment (UP)"), смещенным на фиксированное количество строк для сохранения структуры при последующей автоматической сборке в мастер-таблицу.

Все этапы выполнения сопровождаются логгированием в консоль с указанием объемов входных и выходных данных. Ошибки доступа к файлам или сбои сетевых API изолируются внутри блоков tryCatch, что позволяет циклу продолжать непрерывную обработку пула из 35 файлов даже при возникновении локальных аномалий данных.
